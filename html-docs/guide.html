User Guide

Everything you need to know to use PrimeEdit effectively — from first launch to power features.

Installation

Prerequisites

PrimeEdit requires Qt 6 (or Qt 5) development libraries and CMake.

# Ubuntu / Debian
sudo apt install qt6-base-dev qt6-tools-dev qt6-svg-dev cmake build-essential

# Fedora
sudo dnf install qt6-qtbase-devel qt6-qtsvg-devel cmake gcc-c++

# Arch Linux
sudo pacman -S qt6-base qt6-svg cmake base-devel

Build & Run

git clone <repository-url> prime-edit
cd prime-edit
mkdir build && cd build
cmake ..
make -j$(nproc)
./PrimeEdit
Tip: After building, PrimeEdit automatically installs its icon and desktop launcher to ~/.local/share/. You can find it in your application menu by searching "PrimeEdit".

First Launch

When you open PrimeEdit for the first time:

  • The editor opens with the Notepad++ theme (white background, blue keywords)
  • An empty Untitled 1 tab is ready for typing
  • The toolbar shows common actions as 16x16 icons in a single row
  • The status bar at the bottom shows line/column, encoding, and line ending
Unsaved work is safe. If you close PrimeEdit with unsaved content in any tab — even untitled tabs — everything is silently preserved. Next time you open PrimeEdit, all your tabs reappear exactly as you left them. No prompts. No save dialogs on quit.

Encoding & Line Endings

PrimeEdit auto-detects file encoding (UTF-8, UTF-8-BOM, UTF-16, Windows-1252, ISO-8859, and more). You can convert between encodings and line endings using:

  • The Encoding menu in the menu bar
  • Clicking the encoding indicator in the status bar (e.g., "UTF-8")
  • Clicking the EOL indicator in the status bar (e.g., "LF")

Working with Tabs

  • New tab: Ctrl+N or double-click empty tab bar area
  • Close tab: Ctrl+W, click the X, or middle-click the tab
  • Switch tabs: Ctrl+PgUp / Ctrl+PgDn, or click the tab
  • Reorder: Drag tabs to rearrange
  • Overflow: When too many tabs open, scroll arrows appear

Right-Click Context Menu

Right-click any tab for options: Close, Close Others, Close to the Left, Close to the Right, Close All, Copy Full Path, Open Containing Folder.

Tab Icons

Each tab shows a colored icon based on file type — blue for C/C++, green for Python, yellow for JavaScript, etc. Modified files show a red indicator.

Line Operations

Under Edit > Line Operations: sort lines (ascending/descending), remove duplicates, remove empty lines, join lines, trim whitespace. All operations preserve undo history.

Case Conversion

Under Edit > Convert Case: UPPERCASE, lowercase, Title Case, Sentence Case, iNVERT cASE.

Whitespace & Guides

Under View, toggle: Show Whitespace (dots for spaces, arrows for tabs), Show End of Line (pilcrow symbols), Show Indent Guide (vertical dotted lines).

Finding & Replacing

Quick Search

Ctrl+I opens the incremental search bar — a compact bar that highlights matches as you type. Press Enter for next match, Shift+Enter for previous, Escape to close.

Full Search Dialog

Ctrl+F opens the tabbed search dialog with four tabs:

  1. Find — Find Next, Count, Find All in Current Document, Find All in All Open Documents
  2. Replace (Ctrl+H) — Replace one, Replace All, Replace in All Open Documents
  3. Find in Files (Ctrl+Shift+F) — Search across files on disk with directory picker and filters
  4. Mark — Highlight all matches with yellow background. Optionally bookmark marked lines.

Search Modes

  • Normal — literal text matching
  • Extended — recognizes \n (newline), \t (tab), \r (return), \xNN (hex byte)
  • Regular Expression — full regex with backreferences in replace (\1, $1)

Search Results Panel

When you use "Find All", results appear in a bottom panel grouped by file. Double-click any result to jump to that location.

Bookmarks

Bookmarks let you mark lines for quick navigation.

  • Toggle bookmark: Click the bookmark margin (leftmost gutter column) on any line — a blue circle appears
  • Next bookmark: F2
  • Previous bookmark: Shift+F2
  • Clear all: via Search menu

Bookmarks are saved with your session and persist across restarts.

Column Selection

Hold Alt and drag with the mouse to create a rectangular (column) selection. A semi-transparent blue rectangle shows the selected area.

  • Type to insert text at every line within the selection
  • Escape to cancel the selection
  • Column Editor (Alt+C) — opens a dialog to insert text or sequential numbers at the column position

Code Folding

The fold margin (rightmost gutter column) shows fold indicators:

  • [-] — click to fold (collapse) a code block
  • [+] — click to unfold (expand) a collapsed block
  • Fold All / Unfold All — via View menu
  • Folded blocks show [...] inline after the header line

Folding works for brace-based languages (C, C++, Java, JavaScript, Rust, Go) and indent-based languages (Python, YAML). Fold state persists across sessions.

Auto-Completion

  • Manual trigger: Ctrl+Space at any time
  • Automatic trigger: after typing 3 or more word characters, a popup appears
  • Accept: Enter or Tab
  • Dismiss: Escape
  • Navigate: Up/Down arrow keys

Completion Sources

  1. LSP completions (when a language server is active — highest priority)
  2. Language keywords from the syntax definition
  3. Document words (unique words 4+ characters from your current file)

Custom Server Configuration

To add or override language servers, create ~/.config/PrimeEdit/lsp-servers.json:

{
  "python": {"command": "pylsp", "args": []},
  "c++": {"command": "clangd", "args": ["--background-index"]},
  "rust": {"command": "rust-analyzer", "args": []}
}

Macros

Record a sequence of actions and replay them.

  1. Press F9 to start recording
  2. Perform your editing actions (typing, navigation, etc.)
  3. Press Shift+F9 to stop recording
  4. Press F10 to play back the macro

Run Multiple Times

Via Macro > Run Multiple Times: enter a count, or choose "Run until end of file" to repeat the macro until the cursor reaches the last line.

Save & Load

Macros can be saved to disk and loaded later. Saved macros persist across restarts via ~/.config/PrimeEdit/macros.json.

Large Files

PrimeEdit handles files that other editors cannot open.

  • Files >10MB automatically open in large file mode — memory-mapped, viewport-loaded
  • Files >100MB open in read-only mode by default
  • A 1GB file uses only 95MB of memory (verified by stress testing)
  • Scrolling is smooth — only visible lines plus a buffer are in memory
No special action needed. Just open the file normally. PrimeEdit detects the file size and switches modes automatically.

Schema Validation

PrimeEdit can validate documents against rule sets — a capability no other editor has.

How to Use

  1. Create a file called .olive-schema.json in your project root
  2. Open any file in that directory — PrimeEdit finds the schema automatically
  3. Violations appear as inline underlines and in the Validation Panel (bottom dock)
  4. Click a violation to jump to the exact line

Example Schema

{
  "type": "documentation",
  "rules": [
    {
      "name": "no_todo",
      "pattern": "\\bTODO\\b",
      "mustExist": false,
      "message": "TODO items should be resolved before publish",
      "severity": "warning",
      "scope": "line"
    }
  ]
}

You can also trigger validation manually via Tools > Validate Document.

Block Evaluation

Transform any selected text using an LLM endpoint.

  1. Select the text you want to transform
  2. Press Ctrl+Shift+E
  3. The text is sent to your configured endpoint (Ollama by default)
  4. A diff view appears showing additions (green) and removals (red)
  5. Press A to accept the result into your document, or Escape to reject

Configure Endpoint

Via Tools > Configure Endpoint or edit ~/.config/PrimeEdit/eval-endpoint.json:

{
  "name": "Ollama",
  "url": "http://localhost:11434/api/generate",
  "model": "llama3",
  "type": "ollama",
  "systemPrompt": "Improve the following text. Return only the improved text."
}

Supports both Ollama and OpenAI-compatible endpoints.

Islamic Knowledge Integration

For scholars working on Islamic treatises, PrimeEdit integrates with a ChromaDB-backed RAG system.

Hadith Lookup

Select a hadith reference and press Ctrl+Shift+H. PrimeEdit queries the knowledge base and returns: grading (sahih/hasan/da'if), source collection, and grading authority.

Automatic Activation

When PrimeEdit detects a yameen-treatise file (by extension .treatise/.hadith, or presence of manuscript.yaml/treatise.yaml), it automatically activates the Islamic hadith schema and connects to the ChromaDB bridge.

Arabic RTL

Arabic text renders right-to-left automatically. PrimeEdit detects Arabic characters (U+0600-U+06FF) per block and sets the layout direction. Font fallback includes Noto Sans Arabic and Amiri.

Changing Themes

Open Settings > Preferences, go to the Appearance tab, and select a theme from the dropdown. The change applies instantly — including syntax highlighting colors.

Available Themes

Notepad++ (default), Default Light, Default Dark, Olive, Monokai, Dracula, Nord, Solarized Light, Solarized Dark.

Custom themes can be created as JSON files and imported via the Theme Editor.

Settings & Preferences

Open via Settings > Preferences or Ctrl+, (on some platforms).

Editor Tab

  • Font family — choose any monospace font (default: DejaVu Sans Mono)
  • Font size — 6pt to 72pt
  • Tab width — 1 to 16 spaces
  • Insert spaces — tabs vs spaces
  • Word wrap, line numbers, highlight current line, minimap

Files Tab

  • Default encoding and line ending
  • Trim trailing whitespace and insert final newline on save
  • Auto-save interval (default: 30 seconds)
  • Restore session on startup
Settings are saved to ~/.config/PrimeEdit/settings.json and take effect immediately. Click Apply to see changes without closing the dialog.
