PrimeEdit

A native editor for structured documents, where content is validated, enforced, and executable. Built for performance and extensibility on Linux.

PrimeEdit is a high-performance, native text editor purpose-built for Linux. It combines the editing power of Notepad++ with structured document validation, LLM-powered block evaluation, and Islamic scholarly knowledge integration — capabilities no other editor offers.

Positioning: PrimeEdit is not an IDE. It is not an AI assistant. It is the editor for people doing structured, rule-bound, high-stakes text work on Linux.

Core Principles

  • Performance first — sub-second startup, 1GB+ file handling, 60fps scrolling
  • Structured validation — documents have rules; PrimeEdit enforces them
  • Execution, not assistance — block evaluation sends text to endpoints and returns results inline
  • Domain-aware — Islamic scholarly workflows are first-class citizens

02 Building from Source

Prerequisites

# Ubuntu/Debian (Qt6)
sudo apt install qt6-base-dev qt6-tools-dev qt6-svg-dev cmake build-essential

# Fedora (Qt6)
sudo dnf install qt6-qtbase-devel qt6-qtsvg-devel cmake gcc-c++

# Arch Linux (Qt6)
sudo pacman -S qt6-base qt6-svg cmake base-devel

Build & Run

git clone <repository-url> prime-edit
cd prime-edit
mkdir build && cd build
cmake ..
make -j$(nproc)
./PrimeEdit

Install

sudo make install
Optional: Install ICU libraries (libicu-dev) for extended encoding support (Windows-1252, Shift-JIS, GB2312, Big5, KOI8-R).

05 Code Folding

Block-caching fold implementation (reliable on Qt5 and Qt6):

  • Brace-based folding for C-like languages (C, C++, Java, JavaScript, Rust, Go, etc.)
  • Indent-based folding for Python, YAML, and similar languages
  • Fold All / Unfold All via menu or keyboard
  • Fold level controls (Alt+1 through Alt+8)
  • Folded regions show [...] inline indicator
  • Fold state persists across sessions

Column Editor Dialog

  • Text mode: Insert text at column position across all selected lines
  • Number mode: Insert sequential numbers with configurable start, step, and leading zeros

07 Large File Handling

PrimeEdit handles files that other editors cannot open. Files over 1GB are first-class citizens.

MetricTargetMechanism
Open 1GB file< 2 secondsMemory-mapped I/O with line index
Scroll 500MB fileNo frame drops > 16msViewport loading with buffer hysteresis
Regex search 1GB< 5 seconds1MB chunk processing with progressive results
Memory at 1GB< 200MB RSSOnly visible lines + 500-line buffer in memory
Syntax highlight 10MBNo UI blockProgressive: only viewport is highlighted

How It Works

  • Files >10MB trigger large file mode automatically
  • File is memory-mapped via QFileDevice::map() — no full load into RAM
  • A line-offset index is built on open (~1s for 1GB)
  • Only visible lines plus a 500-line buffer are loaded into the editor
  • Scrollbar maps to the line index for instant navigation
  • Files >100MB open in read-only mode by default

Benchmark Harness

./benchmarks/bench_largefile.sh ./build/PrimeEdit

Generates 100MB, 500MB, and 1GB test files and measures open time and RSS memory.

09 Find in Files

Search across files on disk with directory picker, semicolon-separated glob filters (e.g., *.cpp;*.h), recursive subdirectory option, and hidden folder inclusion.

Results appear in the Search Results Panel (bottom dock), grouped by file. Double-click any result to open the file and navigate to the match.

For large files, PrimeEdit uses threaded chunk-based search with progressive result streaming and progress reporting.

10 Mark & Highlight

The Mark tab in the search dialog highlights all matches with a persistent yellow background. Optionally bookmarks all marked lines. Clear All Marks removes highlights.

11 Syntax Highlighting

Data-driven syntax highlighting powered by JSON definition files. No hardcoded language rules.

How It Works

  • Each language is defined in a resources/syntax/<language>.json file
  • Definitions include: keywords, types, builtins, comment patterns, string delimiters, number patterns, function patterns, preprocessor prefix
  • The SyntaxHighlighter reads the JSON and generates regex rules at runtime
  • Colors come from the active Theme — switching themes updates syntax colors live
  • Language is auto-detected from file extension, filename, shebang line, or content heuristics

Adding a New Language

Create a JSON file in resources/syntax/:

{
  "name": "MyLanguage",
  "keywords": ["if", "else", "for", "while", "return"],
  "types": ["int", "string", "bool"],
  "singleLineComment": "//",
  "multiLineCommentStart": "/*",
  "multiLineCommentEnd": "*/",
  "stringDelimiters": ["\"", "'"],
  "numberPattern": "\\b[0-9]+\\b",
  "functionPattern": "\\b([a-zA-Z_]\\w*)\\s*\\(",
  "foldBased": "brace"
}

Register in resources/resources.qrc and add extension mappings in languagemanager.cpp.

14 Auto-Completion

Two completion sources, with LSP taking priority when available:

  • LSP completions — when a language server is active, completions come from the server
  • Keyword + document words — fallback: language keywords from JSON definitions + unique words (4+ chars) from open documents

Triggers: Ctrl+Space (manual) or automatically after typing 3+ consecutive word characters.

15 Encoding & Detection

Heuristic Charset Detection

PrimeEdit detects file encoding automatically using a multi-stage approach:

  1. BOM detection — UTF-8-BOM, UTF-16 LE/BE, UTF-32 LE
  2. UTF-8 validation — if all bytes decode cleanly, assume UTF-8
  3. Byte-frequency heuristics — Windows-1252 (0x80-0x9F range), ISO-8859-1, Shift-JIS (lead byte patterns), GB2312, Big5

Supported Encodings

Always available: UTF-8, UTF-8-BOM, UTF-16 LE/BE, UTF-32 LE/BE, Latin-1. With ICU: Windows-1252, ISO-8859-2 through 15, KOI8-R/U, Shift-JIS, EUC-JP, GB2312/GBK/GB18030, Big5.

Status Bar Integration

Encoding and line ending indicators in the status bar are clickable — click to convert between encodings or line endings.

16 Persistent Unsaved Documents

Signature feature: This is the Notepad++ behavior users love most. Untitled and modified documents survive application close without prompting.
  • Untitled tabs with content are silently serialized to ~/.local/share/PrimeEdit/sessions/unsaved/
  • Named files with unsaved changes have their modified content cached alongside the file path
  • No save prompt on quit — everything is persisted. The prompt only appears for explicit tab close (Ctrl+W)
  • Recovery dialog on startup if crash-recovery files are detected
  • Auto-recovery timer saves recovery data every 5 seconds

17 File Change Monitoring

PrimeEdit watches all open files via QFileSystemWatcher. When a file is modified externally, a yellow notification bar appears:

The file "main.cpp" has been modified by another program. Do you want to reload it?   [Yes]   [No]

Reload from Disk (Ctrl+Shift+R) also available manually via File menu.

18 Document Schema Validation Unique

The feature no other editor has. Linting for structured documents with enforced rules.

How It Works

  1. Place a .olive-schema.json file in your project root
  2. PrimeEdit detects it automatically when opening files in that directory (searches up to 5 parent levels)
  3. Rules are validated against the document content in real-time
  4. Violations appear as inline underlines and in the Validation Panel

Schema Format

{
  "type": "hadith",
  "rules": [
    "must_include_grading",
    "must_include_source",
    "no_weak_hadith",
    {
      "name": "custom_rule",
      "pattern": "\\bTODO\\b",
      "mustExist": false,
      "message": "TODO items should be resolved",
      "severity": "warning",
      "scope": "line"
    }
  ]
}

Built-in Rules

RuleScopeSeverityDescription
must_include_gradingBlockErrorHadith citation must include grading (sahih/hasan/da'if)
must_include_sourceBlockWarningCitation must reference source collection (Bukhari, Muslim, etc.)
no_weak_hadithLineWarningFlags weak/fabricated hadith references
must_include_arabicBlockInfoBlock should contain Arabic text
quran_citation_formatDocumentWarningQuranic citations should follow Surah:Ayah format

Validation Panel

Bottom dock showing all violations grouped by severity. Click any violation to navigate to its location. Status bar shows health indicator: All clear or 3 errors, 2 warnings.

19 Block Evaluation with Diff Unique

Select text. Evaluate it against any LLM endpoint. See the result inline with a diff. Accept or reject. The document is the interface.

Distinction: This is not Copilot. Other editors add AI as an assistant layer. PrimeEdit adds AI as an execution layer. The result lives in the document.

Workflow

  1. Select text in the editor
  2. Press Ctrl+Shift+E
  3. Text is sent to the configured LLM endpoint
  4. Result appears in an inline overlay with line-by-line diff (green = additions, red strikeout = removals)
  5. Press A to accept (replaces selection, full undo support) or Escape to reject

Endpoint Configuration

Stored in ~/.config/PrimeEdit/eval-endpoint.json. Configurable via Tools > Configure Endpoint.

20 Islamic Knowledge Integration Moat

The feature that makes PrimeEdit unreplicable. Hadith validation, Quranic verse verification, and Arabic RTL rendering — purpose-built for Islamic scholarly authorship.

📚

Hadith Inline Lookup

Ctrl+Shift+H on a hadith reference validates against the 16,000-chunk RAG system via ChromaDB/FastAPI. Returns grading, source, and authority.

📖

Quranic Verse Validation

Surah/ayah citations checked against the Uthmanic rasm corpus. Returns Arabic text and translation for verification.

ا

Arabic RTL Rendering

Per-block text direction detection. Blocks containing Arabic characters (U+0600-U+06FF) automatically render right-to-left. Font fallback: Noto Sans Arabic, Amiri.

📑

Yameen Pipeline Aware

Auto-detects yameen-treatise files by extension (.treatise, .hadith), directory structure, or config files. Activates Islamic schema and bridge automatically.

ChromaDB Bridge

Connects to a local FastAPI instance (default: http://localhost:8000) providing:

  • /api/hadith/validate — Hadith text validation and grading
  • /api/quran/validate — Quranic citation verification
  • /api/scholar/lookup — Scholar attribution check
  • /health — Connection status

23 Macros & Plugins

Macro System

  • Record: F9 (start), Shift+F9 (stop)
  • Playback: F10
  • Run Multiple Times: Macro menu > dialog with count or "until end of file"
  • Save/Load: Persist macros to files for reuse

Plugin System

  • C++ shared library plugin API (PluginInterface)
  • Event hooks: file opened/saved/closed, text changed, selection changed, cursor moved
  • Custom commands registered by plugins
  • Plugin directories: ~/.config/PrimeEdit/plugins/ and /usr/lib/prime-edit/plugins/
  • Plugins menu with enable/disable toggle
