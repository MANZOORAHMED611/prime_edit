3. Data Flow

File Opening Sequence

1. User: File → Open
2. MainWindow::openFile() prompts for path
3. DocumentManager::openDocument(path)
4. Create new Document instance
5. Document::load() reads file with encoding detection
6. MainWindow::createEditor(document)
7. Create Editor widget with Document reference
8. Editor constructor loads content from Document
9. SyntaxHighlighter attached to Editor's QTextDocument
10. Tab added to TabWidget with Editor
11. StatusBar updated with file info

Text Editing Sequence

1. User types in Editor (QPlainTextEdit)
2. Editor::keyPressEvent() handles special keys (Tab, Enter)
3. Text inserted into QPlainTextEdit's internal QTextDocument
4. QPlainTextEdit::textChanged signal emitted
5. Editor::onTextChanged() marks Document as modified
6. Document::modifiedChanged signal emitted
7. MainWindow updates tab title with "*" indicator
8. SyntaxHighlighter::highlightBlock() called for changed lines
9. Auto-save timer triggered (every 5 seconds)
10. Document::saveRecovery() saves to recovery location

Session Restore Sequence

1. Application starts
2. Settings::instance().load() reads settings.json
3. If restoreSession enabled:
4.   Session::restore(mainWindow)
5.   Read session.json for window geometry and file paths
6.   For each saved file path:
7.     MainWindow::openFile(path)
8.   Restore window position and size
9.   Set active tab index

4. Design Patterns Used

Singleton Pattern

  • Settings::instance() - Global settings access
  • DocumentManager::instance() - Central document registry
  • Session::instance() - Session management

Observer Pattern (Qt Signals/Slots)

  • Document emits modifiedChanged → MainWindow updates tab
  • Settings emits fontChanged → Editor updates font
  • Editor emits cursorPositionChanged → StatusBar updates

Model-View Pattern

  • Document (Model) ↔ Editor (View/Controller)
  • QFileSystemModel (Model) ↔ QTreeView in FileExplorer (View)

Command Pattern (via Piece Table)

  • Insert/Remove operations stored in undo stack
  • Each operation is reversible
  • Undo/redo implemented as stack manipulation

5. Memory Management

Ownership Model

Object Owner Lifetime
Document instances DocumentManager (via Qt parent-child) Until explicitly closed
Editor widgets TabWidget (via Qt parent-child) Until tab closed
SyntaxHighlighter Editor (via Qt parent-child) Same as Editor
LineNumberArea Editor (via Qt parent-child) Same as Editor
Dialogs (Find, Preferences) MainWindow (via Qt parent-child) Application lifetime
Qt Memory Model: Objects with parent are automatically deleted when parent is destroyed. Manual deletion is only needed for objects without parents.

Piece Table Memory Efficiency

  • Original Buffer: Stores initial file content (never modified)
  • Add Buffer: Stores all inserted text (append-only)
  • Piece Descriptors: Small structs pointing to buffers
  • Memory Growth: Linear with edits, not file size
  • Example: 1MB file + 100KB edits = ~1.1MB memory (vs 2MB+ for gap buffer)

6. Threading Model

Current Implementation (Single-Threaded)

All operations currently run on the main Qt event loop thread. This is sufficient for Phase 1-2 features and typical file sizes (< 100MB).

Future Multi-Threading (Phase 3+)

Operation Thread Strategy Benefit
Large file loading QThread worker Non-blocking UI
Find in files QThread pool Parallel search across files
LSP communication QProcess with async I/O Non-blocking language server queries
Syntax highlighting Main thread (incremental) Direct QTextDocument access required

8. Error Handling Strategy

File I/O Errors

  • Use QFile::errorString() for detailed messages
  • Show QMessageBox with error details to user
  • Graceful fallback (e.g., don't crash if recovery file missing)

Encoding Errors

  • Attempt detection using BOM or heuristics
  • Fallback to UTF-8 if detection fails
  • Allow user to override encoding in status bar

Crash Recovery

  • Auto-save recovery files every 5 seconds
  • Detect abnormal shutdown via lock file
  • Present recovery dialog on next start
  • Allow selective recovery or discard
