19. Plugin Architecture ✓ Complete

Purpose

Enable extensibility through a plugin system that allows third-party developers to add functionality to Olive Notepad.

Components

Component File Description
PluginInterface src/core/plugininterface.h Abstract interface that all plugins must implement
PluginManager src/core/pluginmanager.h/cpp Manages plugin loading, lifecycle, and event broadcasting

Plugin API

Plugins can hook into the following events:

  • onFileOpened(Document*) - Called when a file is opened
  • onFileSaved(Document*) - Called when a file is saved
  • onFileClosed(Document*) - Called when a file is closed
  • onTextChanged(Editor*) - Called when text is modified
  • onSelectionChanged(Editor*) - Called when selection changes
  • onCursorMoved(Editor*) - Called when cursor position changes

Plugin Directories

  • Application directory: ./plugins
  • User directory: ~/.local/share/notepad-supreme/plugins
  • System-wide: /usr/share/notepad-supreme/plugins
  • Local system: /usr/local/share/notepad-supreme/plugins

Custom Commands

Plugins can register custom commands accessible via the command palette using the format PluginName::CommandName.

Built-in Themes

  • Default Dark - VS Code dark inspired theme
  • Default Light - VS Code light inspired theme
  • Monokai - Classic Monokai by Wimer Hazenberg
  • Dracula - Popular Dracula theme by Zeno Rocha
  • Nord - Arctic Ice Studio's Nord color palette
  • Solarized Dark - Ethan Schoonover's Solarized dark
  • Solarized Light - Ethan Schoonover's Solarized light

Color Categories

Themes define colors for:

  • Editor - Background, foreground, selection, current line, line numbers
  • Syntax - Keywords, strings, comments, numbers, functions, types, variables, operators
  • UI - Menus, toolbars, status bar, tabs
  • Accents - Primary, secondary, borders
  • Diagnostics - Errors, warnings, info, hints

Theme Editor Features

  • Live preview of theme changes
  • Color picker for all 30+ colors
  • Save/load custom themes
  • Import/export themes as JSON
  • Metadata editing (name, author, dark/light)

23. Remote File Editing (SFTP/SSH) ✓ Complete

Purpose

Edit files on remote servers via SFTP/SSH without leaving the editor.

Components

Component File Description
RemoteConnection src/core/remoteconnection.h/cpp SSH/SFTP/SCP connection handling
RemoteFileManager src/core/remoteconnection.h/cpp Manages multiple connections and file mappings
RemoteConnectionDialog src/ui/remoteconnectiondialog.h/cpp UI for managing remote connections

Supported Protocols

  • SFTP - Secure File Transfer Protocol
  • SSH - Secure Shell for command execution
  • SCP - Secure Copy Protocol

Features

  • Save/load connection profiles
  • Multiple simultaneous connections
  • Download file, edit locally, upload on save
  • Directory listing and browsing
  • File operations (delete, create directory)
  • Progress tracking for uploads/downloads

Implementation

Uses command-line tools ssh and scp via QProcess, with temporary local files for editing.

24. Enhanced Git Integration ✓ Complete

Purpose

Visual diff viewing and interactive staging for Git repositories.

Components

Component File Description
GitDiffViewer src/ui/gitdiffviewer.h/cpp Visual diff viewer with staging controls
GitStagingDialog src/ui/gitdiffviewer.h/cpp Complete staging workflow with commit

Features

  • Diff Viewing - Color-coded diff with syntax highlighting
  • File Tree - Organized view of staged/unstaged changes
  • Interactive Staging - Stage/unstage individual files
  • Discard Changes - Revert modifications to files
  • Commit - Create commits with message input
  • Status Indicators - Modified, added, deleted, renamed

Diff Colors

  • Green - Added lines (+)
  • Red - Deleted lines (-)
  • Dark Blue - File headers (+++/---)
  • Cyan - Hunk headers (@@)

Git Commands Used

  • git diff - Show unstaged changes
  • git diff --cached - Show staged changes
  • git add - Stage files
  • git reset HEAD - Unstage files
  • git checkout -- - Discard changes
  • git commit -m - Create commit

Technical Details

New Dependencies

  • Qt6::Network (for remote connections)
  • External tools: ssh, scp, git

Design Patterns

  • Singleton - ThemeManager, PluginManager, RemoteFileManager
  • Observer - Plugin event broadcasting
  • Strategy - Different connection types (SFTP/SSH/SCP)
  • Factory - Built-in theme creation
