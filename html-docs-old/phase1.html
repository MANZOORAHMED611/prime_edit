Technical Implementation:

Document* DocumentManager::openDocument(const QString &filePath)
{
    // Check if already open
    for (Document *doc : m_documents) {
        if (doc->filePath() == filePath) {
            return doc;  // Return existing document
        }
    }

    // Create and load new document
    Document *doc = new Document(this);
    if (doc->load(filePath)) {
        m_documents.append(doc);
        emit documentCreated(doc);
        return doc;
    }

    delete doc;
    return nullptr;
}

Components Involved:

  • MainWindow - UI and menu actions
  • DocumentManager - Document lifecycle management
  • Document - File I/O and metadata
  • TabWidget - Multi-document interface

    // Extension mapping
    if (ext == "cpp" || ext == "cc" || ext == "cxx") return "C++";
    if (ext == "py") return "Python";
    if (ext == "js") return "JavaScript";
    if (ext == "rs") return "Rust";

Auto-Detection:

Document::LineEnding Document::detectLineEnding(const QString &text)
{
    int crlfCount = text.count("\r\n");
    int lfCount = text.count('\n') - crlfCount;
    int crCount = text.count('\r') - crlfCount;

    if (crlfCount > lfCount && crlfCount > crCount)
        return Windows;
    if (crCount > lfCount)
        return ClassicMac;
    return Unix;  // Default to Unix
}

6. Unlimited Undo/Redo

Piece Table Implementation

Features:

  • ✓ Unlimited undo history (memory permitting)
  • ✓ Redo support
  • ✓ Atomic operations (grouped edits)
  • ✓ Efficient memory usage via piece table

How It Works:

Uses a Piece Table data structure for efficient text editing with undo/redo support. Instead of modifying the original text, operations are recorded as pieces pointing to two buffers:

  • Original Buffer: Never modified, stores initial file content
  • Add Buffer: Append-only, stores all inserted text
  • Piece Descriptors: List of {source, start, length} describing the document

Example:

Original: "Hello World"
User inserts "Beautiful " at position 6

Piece Table:
  [Original, 0, 6]     -> "Hello "
  [Add, 0, 10]         -> "Beautiful "
  [Original, 6, 5]     -> "World"

Result: "Hello Beautiful World"

Undo: Simply remove middle piece
Redo: Add middle piece back

Advantages:

  • O(1) undo/redo operations
  • Memory efficient - only stores deltas
  • Original file preserved for quick undo-all
  • Supports atomic multi-operation edits

8. Crash Recovery System

Automatic Backup

Features:

  • ✓ Auto-save every 5 seconds (configurable)
  • ✓ Recovery files stored in ~/.local/share/notepad-supreme/recovery/
  • ✓ Detects abnormal shutdown (crash detection)
  • ✓ Recovery dialog on startup after crash
  • ✓ Selective file recovery

Recovery File Format:

document_.json:
{
  "filePath": "/home/user/file.txt",
  "content": "...",
  "encoding": "UTF-8",
  "lineEnding": "Unix",
  "language": "Python"
}

Process:

  1. Every 5 seconds, DocumentManager checks for modified documents
  2. For each modified document, saves recovery file with hash-based filename
  3. On normal shutdown, recovery files are deleted
  4. On crash/abnormal shutdown, recovery files remain
  5. Next startup detects recovery files and offers to restore
  6. User can choose to restore all, restore selected, or discard

Safety Features:

  • Original files never overwritten automatically
  • Recovered content marked as modified (requires explicit save)
  • Recovery files cleaned up after successful restoration
  • Handles multiple instances gracefully
